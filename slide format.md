Epic Name

Performance Enhancement

Owners

•Product Team
•Frontend Development Team
•Backend Development Team
•Quality Assurance
•
Objectives

•Transition from a pull-based update mechanism to a push-based system using Firebase.
•Reduce unnecessary loaders and improve the speed of document retrieval and page transitions.
•Enhance app stability and responsiveness for a smoother UX/UI.
•
Description

This epic aims to remove redundant loading times and optimize the user experience by implementing Firebase push updates instead of the current pull-based mechanism. By reducing the need for repeated data fetches every time a user visits a page or opens a document, the app will provide a more seamless and responsive experience.

Dependencies 

•Firebase integration for real-time updates.
•API adjustments to support push-based mechanisms.
•Performance testing to measure improvements in load times.
•
Acceptance Criteria

•Implement Firebase push updates for document-related actions.
•Eliminate redundant API calls that cause multiple loading screens.
•Reduce document loading times and improve navigation fluidity.
•Ensure smooth transition between pages without unnecessary delays.
•
Timeline

•Start date: December 2024
•End date: February 2025
Target Audience

•UAE PASS Users
•Service Providers
