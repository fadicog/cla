**Why:** The entire 2026 roadmap is grounded in analysis of 350,802 sharing requests from Nov 12-18, 2025. These numbers inform every prioritization decision. Note: all figures are estimates pending Status-Based Reporting going live.

**How to apply:** When the PM asks about "why" a feature is prioritized, these data points are the answer. Also use to ground conversations about targets and success criteria.

## Headline Metrics (Estimated Baselines)
| Metric | Value |
|--------|-------|
| Overall sharing conversion rate | 67.4% |
| Success when docs available | 84.9% |
| Success when docs missing | 0.0% |
| "Dead on arrival" requests | 20.6% (SPs requesting docs users don't have) |
| Technical failure rate | 3.5% |
| User abandonment rate | 17.8% |
| iOS conversion | 77.8% |
| Android conversion | 67.7% |
| iOS-Android gap | ~10% |
| Consent screen drop-off | 16.9% (largest single funnel leak) |
| Weekly sharing requests analyzed | ~350K |

## Three Critical Problems
1. 72K futile requests/week — SPs requesting documents users do not have
2. 28K abandonments/week despite docs being available — UX/engagement issue
3. 15K lost shares/week from Android gap — platform-specific

## Status-Based Reporting System (23 Status Codes)
Designed to enable accurate measurement. Code ranges:
- 100-199: Pending/In-Progress
- 200-299: Success states
- 300-399: User abandonment
- 400-499: Document/issuer failures
- 500-599: Technical/system failures
- 600: Unknown/unmapped legacy

Critical dependency: all 2026 conversion targets are estimates until this system is live and baselines re-validated.

## Top Recommendations (from Nov 2025 analysis)
1. Document Pre-Check API — eliminate 72K futile requests/week
2. Consent Screen UX Redesign — reduce 16.9% drop-off
3. Android Optimization Sprint — close 10% platform gap
4. Issuer Retry Logic — reduce technical failures

Potential impact: +31,500 shares/week (+13.3%) -> target 76% conversion (earlier estimate; 80% is the 2026 target)
