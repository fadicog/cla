1 User clicks button on SP website
2 SP backend creates request via API
3 Push notification sent to user's phone

Assessment Verdict

The UX improvement is strategically sound and aligns with CBUAE's push toward app-based auth (Notice 3057). It eliminates QR tampering risk flagged by Dubai Electronic Security Center. However, it shifts the attack surface from physical proximity to remote initiation. All 7 risks are mitigatable with industry-standard controls. Proceed with mitigations as mandatory gates.

Required Mitigations & Recommendation

Mandatory Controls, Risk Comparison, and Decision Framework
SLIDE 3/3 FEB 2026

Must-Have (Launch Blockers)

1

Number Matching / Challenge Code

Display code on SP website; user confirms same code in UAE PASS app. Industry standard since Microsoft/Okta mandate (2023).

2

Aggressive Rate Limiting

Max 3 requests/EID/hour, 5/day. Cool-down + CAPTCHA after failures. Per-IP and device fingerprint throttling.

3

Generic Responses (Anti-Enumeration)

"If details are correct, you'll receive a request." Same response for valid/invalid. Constant-time API responses.

4

Short TTL + Single-Use Tokens

Requests expire in 3–5 min. Non-reusable correlation IDs. Already enforced via DB unique constraint.

Should-Have (Post-Launch)

5

Data Minimization & Retention Policy

Hash EID+phone after linking. Delete raw data within 24hrs. Audit trail for UAE PDPL compliance.

6

Bot Protection & Abuse Monitoring

WAF rules, device attestation, anomaly scoring by IP/ASN/geo. Real-time alerting dashboard.

7

"Report Suspicious Request" + User Muting

Report/block requests from specific SPs. In-app: "We will never call and ask you to approve."

CBUAE Regulatory Tailwind

CBUAE Notice 3057 mandates all UAE financial institutions phase out SMS OTP by March 2026. UAE is the first country globally to take this step. This creates natural demand for the "Continue with UAE PASS" pattern from every bank, insurer, and payment provider.
